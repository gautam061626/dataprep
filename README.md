# 📊 DataPrep Studio

> **DataPrep Studio** is a complete, production-ready web application that looks and feels like a professional, information-dense desktop software from the early 2000s (inspired by SQL Server Management Studio 2008, Microsoft Access 2003, and Windows XP Explorer), but runs on a modern, high-performance **FastAPI** and **Pandas / Scikit-Learn** backend.

---

## 🖥️ Project Overview

Instead of typical minimalist SaaS dashboards with floating cards and gradients, **DataPrep Studio** embraces a high-density classic desktop application layout using the **Windows XP/2000 Tahoma** design system:
- **Zero Page Reloads**: Runs as a single-page application (SPA).
- **Tahoma Typography**: 11px font layout with pixel-aligned 1px/2px outset and inset borders.
- **Visual Skin Themes**: Switch seamlessly between **Windows XP Blue**, **Classic Gray (Windows 2000)**, **Olive Green**, and **Metallic Silver**.
- **Interactive Data Grid**: Direct double-click cell editing, column sorting, pagination, and data search.

---

## 🚀 Features

### 1. File Parsing & Validation
* Upload CSV files with customizable delimiter inputs (Comma, Semicolon, Tab) and header row toggles.
* Load a pre-compiled Customer Churn sample dataset instantly for testing.

### 2. High-Performance Diagnostics & Profiling
* **Metadata Profile**: Automatically infers column datatypes (Numeric, Categorical, Date) using smart sampling.
* **Property Grid**: Displays counts, unique tallies, missing rates, standard deviation, skewness, variance, and quantiles (Q1, Q3) with sub-second execution speeds on large files.
* **Value Distribution**: View category frequency lists and details in real-time.

### 3. Data Cleaning Suite
* **Deduplication**: Purge duplicate rows instantly.
* **Null Imputation**: Fill missing values using mean, median, mode, or custom placeholder strings.
* **Structural Changes**: Drop columns, rename headings, and replace occurrences of specific text strings.
* **String Standardization**: Apply trimming and casing operations (UPPERCASE, lowercase, Title Case).
* **Outliers Removal**: Remove outliers based on standard deviation IQR thresholds.
* **Interactive Timeline Logs**: Real-time logging of all cleaning operations.
* **Undo / Redo / Reset**: Restore previous edits using a dynamic memory-capped history state stack.

### 4. Interactive Plotly Canvas
* Construct dynamic Plotly.js charts (Bar, Pie, Scatter, Histogram, Line, Box plots, and Correlation Heatmaps).
* Apply filters on categorical fields and download clean, static plot images (PNG) generated by Matplotlib.

### 5. Automated Audit Reports
* **Live HTML Preview**: View a structured audit document showing data quality index ratings, schema distributions, and cleaning logs.
* **Print PDF**: Expose a print-ready audit document generated via **ReportLab** containing cover sheets, metrics tables, and formal signature sign-offs.

### 6. Machine Learning Sandbox
* Choose from multiple models: **Linear/Ridge Regression, Decision Trees, KNN, Naive Bayes, Random Forests, and Support Vector Machines (SVM)**.
* Configure train-test split percentages and hyperparameter values.
* Review performance diagnostic tables (Accuracy, Precision, Recall, F1 for Classification; MSE, MAE, R2 for Regression).
* Inspect Evaluation Plot diagnostics, Feature Importance bars, and **SHAP Value Explanations** to check feature impact.

## 🔧 Installation & Local Setup

### Prerequisite
Ensure Python 3.10+ is installed on your local computer.

### Step 1: Clone the Repository
```bash
git clone https://github.com/your-username/dataprep-studio.git
cd dataprep-studio
```

### Step 2: Launch the App
DataPrep Studio is immediately usable after cloning. 

**On Windows:**
Simply double-click the `run.bat` helper script. It will install all backend requirements and start the local server.

**On macOS/Linux:**
```bash
# Install dependencies
pip install -r requirements.txt

# Start the server
uvicorn backend.main:app --host 127.0.0.1 --port 8000
```

### Step 3: Access in Browser
Navigate to **[http://127.0.0.1:8000](http://127.0.0.1:8000)** to launch the application.

## ☁️ Cloud Deployment (Render/Railway)
This project is configured out-of-the-box for cloud hosting:
* **Procfile**: Points to uvicorn with binding arguments (`--host 0.0.0.0 --port $PORT`).
* **runtime.txt**: Sets the default python runtime to `python-3.11.9`.
* Static folders (`frontend/`, `components/`, `pages/`) are mounted relative to the root directory, preventing any file resolution crashes in cloud runner contexts.
